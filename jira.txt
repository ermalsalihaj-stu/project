## TASK 18

**Title:** Implement Jira Integration for Bundle Ingestion
**Assignee:** Intern 1
**Status:** To Do

**Description:**
Create a Jira integration module that fetches issues from Jira using REST API and converts them into a valid local Product Bundle that can be used by the existing pipeline with `python main.py run --bundle ...`.

**Acceptance Criteria:**

* Create `integrations/jira/` with at least:

  * `client.py`
  * `ingest.py`
* Implement `JiraClient` using `requests`
* Implement `search_issues(jql, max_results, start_at)` to retrieve Jira issues
* Implement `ingest_jira_to_bundle(...)`
* Generated bundle folder includes:

  * `bundle_manifest.json`
  * `tickets.json`
* `tickets.json` maps Jira issue fields into internal ticket structure:

  * id/key
  * title/summary
  * description
  * priority
  * labels/tags
  * status
  * created/updated
  * url
* Optional support for generating `customer_notes.md` from comments
* Add automated test using mock Jira response
* Test verifies bundle creation and `tickets.json` generation
* Output bundle works with existing pipeline run command

**Technical Notes:**

* Use env vars:

  * `JIRA_BASE_URL`
  * `JIRA_EMAIL`
  * `JIRA_API_TOKEN`
  * `JIRA_JQL`
* Do not commit secrets
* Suggested branch: `feature/jira-ingest-to-bundle`

## TASK 19

**Title:** Implement GitHub PR Comment/Review Poster for Run Outputs
**Assignee:** Intern 2
**Status:** To Do

**Description:**
Create a GitHub integration that reads pipeline run outputs and automatically posts a summary comment or review to a pull request, including recommendation summary, risks, next steps, and generated artifacts.

**Acceptance Criteria:**

* Create `integrations/github/` with at least:

  * `pr_poster.py`
* Implement `build_pr_message(run_dir)`
* `build_pr_message(run_dir)` reads:

  * `final_recommendation.json`
  * optionally `schema_validation.json`
* Generated PR message includes:

  * Executive summary
  * Gating decision
  * Top risks
  * Suggested next steps
  * List of generated artifacts:

    * `prd.md`
    * `roadmap.json`
    * `experiment_plan.md`
    * `decision_log.md`
    * `backlog.csv`
* Implement `post_pr_comment(repo, pr_number, body, token)`
* Default implementation posts GitHub Issue Comment to PR
* Optional bonus: implement PR review comment mode
* Add automated test with mocked GitHub API request
* Test verifies POST request is sent to correct endpoint

**Technical Notes:**

* Use env vars:

  * `GITHUB_TOKEN`
  * `GITHUB_REPO`
  * `GITHUB_PR_NUMBER`
* Prefer issue comment first because it is simpler and more reliable for demo
* Suggested branch: `feature/github-pr-poster`

## TASK 20

**Title:** Integrate Bonus Features into CLI and End-to-End Demo Flow
**Assignee:** Intern 3
**Status:** To Do

**Description:**
Connect the Jira ingestion flow and GitHub PR posting flow into the existing CLI so the full bonus workflow can be demonstrated end-to-end: ingest Jira issues, generate a bundle, run the pipeline, and post results back to a GitHub PR.

**Acceptance Criteria:**

* Add `requests` dependency to `requirements.txt`
* Extend `cli/app.py` with new subcommands:

  * `ingest-jira`
  * `post-pr`
* `ingest-jira` supports arguments such as:

  * `--jql`
  * `--out`
  * `--project`
* `post-pr` supports arguments such as:

  * `--repo`
  * `--pr`
  * `--run-dir`
  * `--mode comment|review`
* CLI correctly calls Intern 1 Jira ingestion module
* CLI correctly calls Intern 2 GitHub PR posting module
* Add `.env.example` with Jira and GitHub variables
* Update `README.md` with demo commands and setup instructions
* Add end-to-end smoke test with mocks for:

  * Jira response
  * pipeline run
  * GitHub POST
* Demo flow works with these steps:

  1. `python main.py ingest-jira --jql "..." --out bundles/jira_demo`
  2. `python main.py run --bundle bundles/jira_demo --strict`
  3. `python main.py post-pr --repo owner/repo --pr 12 --run-dir runs/<id> --mode comment`
* Optional bonus: add GitHub Action workflow for PR commenting

**Technical Notes:**

* Keep tests independent from real credentials
* Use mocks for Jira and GitHub API calls
* Optional workflow file:

  * `.github/workflows/aipm_pr_comment.yml`
* Suggested branch: `feature/cli-bonus-glue`
